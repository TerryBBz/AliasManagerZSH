# 📦 Projet : Alias Manager – CLI pour ZSH

## 🎯 Objectif
Créer un outil CLI appelé **alias-manager** qui permet de **gérer des alias de terminal** dans **ZSH** de manière simple, organisée et puissante.  
Le but est d’ajouter, lister, supprimer, désactiver ou réactiver des alias sans avoir à éditer manuellement le fichier `.zshrc`.

---

## 🛠️ Fonctionnalités principales

### **1. Ajout d’alias**
```
alias-manager add <nom> "<commande>" [--group nom_du_groupe]
Par défaut, l’alias est ajouté dans le groupe default.

2. Liste des alias



alias-manager list
→ Affiche tous les groupes et alias actifs.




alias-manager list group
→ Affiche uniquement les groupes existants.




alias-manager list <nom_du_groupe>
→ Affiche uniquement les alias actifs de ce groupe.

3. Suppression d’alias



alias-manager remove <nom> [--group nom_du_groupe]
4. Désactivation et réactivation
Désactiver un alias



alias-manager disable <nom> [--group nom_du_groupe]
Réactiver un alias



alias-manager enable <nom> [--group nom_du_groupe]
Désactiver un groupe entier



alias-manager disable-group <nom_du_groupe>
Réactiver un groupe entier



alias-manager enable-group <nom_du_groupe>
Lister les alias désactivés



alias-manager list --show-disabled
5. Sauvegarde et restauration
Sauvegarder tous les alias :




alias-manager backup ~/mes-alias.json
Restaurer depuis un fichier :




alias-manager restore ~/mes-alias.json
📂 Structure de stockage
Les données sont stockées dans :

~/.alias-manager/aliases.json

~/.alias-manager/aliases.sh

Exemple aliases.json :

json


{
  "groups": {
    "default": {
      "serve": { "cmd": "php -S localhost:8000 -t public", "disabled": false }
    },
    "git": {
      "gpl": { "cmd": "git pull origin main", "disabled": false },
      "gcm": { "cmd": "git commit -m", "disabled": true }
    }
  }
}
Exemple aliases.sh généré :




# Generated by alias-manager
# Group: default
alias serve='php -S localhost:8000 -t public'

# Group: git
alias gpl='git pull origin main'
# gcm is disabled
⚙️ Intégration ZSH
Pour charger automatiquement les alias :

Ajouter dans ~/.zshrc :




[ -f "$HOME/.alias-manager/aliases.sh" ] && source "$HOME/.alias-manager/aliases.sh"
Option bonus :

L’outil détecte ZSH et propose d’ajouter cette ligne automatiquement (confirmation demandée).
